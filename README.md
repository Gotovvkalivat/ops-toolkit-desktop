# OPS Toolkit Desktop

Это полноценный рабочий инструмент, а не демонстрационный макет. Расчёт,
оформление, импорт, серверный кеш, лимиты запросов, диагностика и выгрузка
используют те же рабочие модули в обеих поставках.

Единая десктопная оболочка для массового расчёта доставки и массового оформления заказов.

## Что изменилось относительно расширения

- Расчёт и оформление открываются в одной рабочей оболочке и переключаются без перезагрузки.
- Проект, клиент и доступ находятся в общем navbar. Клиенты расчёта и оформления хранятся раздельно.
- Настройки открываются единым правым сайдбаром: общие креды не дублируются, параметры меняются для активного раздела.
- Сетевые запросы, лимиты и кеш выполняются отдельным C#-сервером, поэтому очереди API не блокируют интерфейс.
- Кеш расчёта учитывает проект, учётную запись, клиента, маршрут, тип и параметры груза.
- DaData ограничена 30 запросами в секунду, справочник городов ЛК — 20 запросами в секунду, расчёт — 6 одновременными запросами.
- Запросы выполняются без cookie-контейнера.
- Ошибки получают `traceId`; безопасный журнал доступен в `/api/debug/requests`.
- `/api/rpc` оставлен только для совместимости. Интерфейс использует читаемые REST-маршруты.

## Готовое Windows-приложение

```powershell
.\publish-desktop-app.cmd
```

Готовый файл запуска: `release\OPS-Toolkit-Desktop-App\OPS Toolkit.exe`. Он сам запускает C#-сервер в фоне и показывает интерфейс во встроенном окне WebView2: консоль и отдельная вкладка браузера не открываются. Папку `server` рядом с EXE перемещать или удалять нельзя.

## Полный релиз

```powershell
.\publish-release.cmd
```

Команда собирает две полноценные поставки:

- `release\OPS-Toolkit-Desktop-App` — основное настольное приложение;
- `release\OPS-Toolkit-Localhost-Server` — запуск в обычном браузере через
  localhost без установки Node.js или .NET.

## Запуск сервера для разработки

Нужен .NET 8 SDK:

```powershell
.\start-csharp.cmd
```

Приложение откроется по адресу `http://127.0.0.1:48731/`. Если порт занят, сервер выберет следующий свободный.

Запуск без автоматического открытия окна:

```powershell
.\start-csharp.cmd --no-open
```

Режим расширенной диагностики:

```powershell
.\start-csharp.cmd --debug
```

## Портативная версия для браузера

```powershell
.\publish-portable.cmd
```

Готовая версия появится в `release\OPS-Toolkit-Localhost-Server`. Запускайте
`Start OPS Toolkit.cmd`: браузер откроется автоматически. Поставка включает
.NET Runtime и не требует Node.js, .NET SDK, других сторонних программ или
прав администратора. Сервер не устанавливается как служба и доступен только
на этом компьютере через `127.0.0.1`.

Порядок работы сотрудников описан в `docs\EMPLOYEE_WORKFLOW.md`.

## Диагностика

- `/api/health` — состояние сервера, лимиты и краткая статистика кеша.
- `/api/debug/status` — память процесса, очереди запросов и последние ошибки.
- `/api/debug/self-test` — проверка разделения кеша по клиентам и фильтра транспортного тарифа.
- `/api/debug/cache` — безопасные метаданные кеша без токенов и паролей.
- `/api/debug/requests` — последние запросы, безопасные ответы API и их `traceId`.
- `POST /api/debug/cache/check` — проверка точного попадания расчёта в кеш.

## Основные API-маршруты

- `POST /api/auth/login` — вход в выбранный проект.
- `POST /api/reference/delivery-companies` — справочник ТК.
- `POST /api/clients/search` — поиск клиента по ИНН.
- `POST /api/addresses/resolve` — распознавание адреса.
- `POST /api/tariffs/calculate` — расчёт тарифов.
- `POST /api/orders/create` — создание заказа.
- `POST /api/orders/cancel` — отмена заказа.
- `POST /api/cache/status` и `/api/cache/clear` — проверка и очистка кеша.

## Подготовка Tauri

В `src-tauri` находится оболочка Tauri 2 для следующего этапа. Она запускает опубликованный C#-сервер как sidecar с `--no-open`, поэтому у пользователя останется одно окно без отдельной консоли и браузерной вкладки. Инструкция по размещению sidecar находится в `src-tauri/README.md`.

## Проверка

```powershell
dotnet build .\csharp\OpsToolkit.Desktop.Server\OpsToolkit.Desktop.Server.csproj --no-restore
dotnet build .\csharp\OpsToolkit.Desktop.App\OpsToolkit.Desktop.App.csproj --no-restore
npm run check
```

Node.js остаётся только как временный совместимый dev-host в `server/`; пользователям C#-версии он не нужен.
