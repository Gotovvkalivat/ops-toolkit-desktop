# OPS Toolkit Desktop

Настольная версия OPS Toolkit с локальным фоновым сервером.

## Что внутри

- `server/desktop-server.js` - локальный HTTP-сервер на `127.0.0.1`.
- `server/background-runtime-source.js` - API-логика из расширения: авторизация, DaData, ЛК, расчёт, создание и отмена заказов, кеши, лимиты запросов.
- `public/calculator` - интерфейс массового расчёта.
- `public/orders` - интерфейс массового оформления.
- `desktop-assets/bridge.js` - desktop-bridge: RPC через `/api/rpc` и локальный аналог `chrome.storage.local`.

## Почему должно быть стабильнее расширения

- Запросы в ЛК, DaData, кеширование и лимиты выполняются в отдельном Node-процессе.
- UI не держит сетевую очередь внутри процесса расширения Chrome.
- Кеш сервера живёт отдельно от DOM и интерфейсных перерисовок.
- Запуск идёт в отдельном app-window Chrome/Edge с отдельным профилем:
  `%APPDATA%\ops-toolkit-desktop\browser-profile`.

## Запуск

Вариант 1:

```bat
start.cmd
```

Вариант 2:

```powershell
npm start
```

После запуска откроется окно приложения. Если окно не открылось автоматически, сервер напечатает адрес вида:

```text
http://127.0.0.1:48731/
```

## Проверка

```powershell
npm run check
```

## Обновление интерфейса из расширения

Когда в `ops-toolkit-extension-repo` появляются новые правки UI:

```powershell
npm run sync:ui
```

Скрипт копирует актуальные страницы расчёта/оформления и API-слой, затем возвращает desktop-bridge поверх обычного bridge.

## Важно

Это лёгкая desktop-обёртка без Electron. Ей нужен установленный Node.js 18+ и Chrome или Edge. Вес остаётся маленьким, потому что Chromium не упаковывается внутрь приложения.
